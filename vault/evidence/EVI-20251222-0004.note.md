# 后端技术栈名词清单学习笔记

## Context

在浏览 Java 后端学习路线图、视频教程简介时，看到一串技术名词。目标是快速建立最小分层模型，理解这些名词的位置和相互关系，而不是深入学习每一个。

## Term List

### Runtime Layer（运行时框架）
- **Spring Boot** — Java 应用开发框架，简化配置和部署
- **gRPC** — Google 的高性能 RPC 框架，基于 HTTP/2 和 Protocol Buffers
- **Dubbo** — 阿里开源的 Java RPC 框架，微服务通信

### Data Layer（数据层）
- **MySQL** — 关系型数据库，OLTP 场景
- **PostgreSQL** — 关系型数据库，功能更丰富，扩展性强
- **Redis** — 内存缓存/数据结构存储，热数据加速
- **Elasticsearch** — 全文搜索引擎，日志分析
- **Kafka** — 分布式消息队列，高吞吐，日志/事件流
- **RabbitMQ** — 消息队列，协议丰富，延迟更低

### Infrastructure Layer（基础设施层）
- **Docker** — 容器化，应用打包与隔离
- **Kubernetes (K8s)** — 容器编排，自动伸缩与调度
- **Nginx** — 反向代理/负载均衡/静态资源服务
- **Nacos** — 阿里开源，服务发现 + 配置中心
- **Zookeeper** — 分布式协调服务，服务注册/配置管理

### Delivery Layer（交付层）
- **Git** — 版本控制系统
- **Jenkins** — CI/CD 自动化构建部署

## My Constraints

- **已有基础**: Docker 基础使用、MySQL 基础 SQL、Git 日常操作
- **不熟悉**: 微服务架构、消息队列、容器编排
- **时间预算**: 每天 30 分钟，先建立概念模型
- **目标**: 建立最小分层模型，能回答"为什么需要这个工具"

## 3 Questions

1. **分层模型**: 这些技术如何分层？每层解决什么核心问题？
2. **学习优先级**: 对于想入门后端的人，哪些先学、哪些后学？
3. **边界判断**: 什么场景下需要 Kafka vs RabbitMQ？需要 Nacos vs Zookeeper？
