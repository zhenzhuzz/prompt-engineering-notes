# 观察：Evidence-first 文档方法

日期: 2025-12-22
背景: 搭建知识管理系统

## 观察

在构建知识库时，先采集原始 Evidence（证据）再提出 Claim（主张）可以带来：
1. 更准确的文档
2. 更容易的验证和审计
3. 事实与解读之间更清晰的分离

## 核心洞见

"Evidence-first（证据优先）"方法颠倒了传统工作流：
- 传统方式：先写主张 → 再找支持证据
- Evidence-first：先采集证据 → 再从证据推导主张

这可以防止"claim drift（主张漂移）"——文档逐渐脱离现实的现象。

## 相关概念

- Append-only logs（追加式日志）
- Immutable audit trails（不可变审计追踪）
- Source-of-truth patterns（权威来源模式）
