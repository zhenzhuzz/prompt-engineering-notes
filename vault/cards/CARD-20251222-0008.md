## Claim

对于有 Docker/MySQL/Git 基础的学习者，后端技术栈最小学习路径为：(1) 容器与数据库巩固 → (2) 缓存与消息入门 → (3) 编排与可观测进阶。每阶段用一个最小实验验证理解。

## Evidence

根据 EVI-20251222-0004#loc:constraints，学习者已有 Docker/MySQL/Git 基础，目标是建立概念模型而非精通每个工具。

**阶段规划：**

```
Phase 1: 容器与数据库巩固 (先学)
├── Docker: 写 Dockerfile，理解 image/container/volume
├── MySQL: 索引优化、事务隔离级别
└── 验证: 用 Docker 部署 MySQL + 简单 API

Phase 2: 缓存与消息入门 (次学)
├── Redis: 数据结构、缓存穿透/击穿/雪崩
├── Kafka 或 RabbitMQ: 选一个，理解 pub/sub 模式
└── 验证: API 加 Redis 缓存 + Kafka 写日志

Phase 3: 编排与可观测进阶 (后学)
├── Kubernetes: Pod/Service/Deployment 概念
├── Nginx: 反向代理配置
└── 验证: K8s 部署多副本 + Nginx 负载均衡
```

**暂缓学习（触发条件明确后再学）：**
- gRPC/Dubbo: 当有多服务通信需求时
- Nacos/Zookeeper: 当有服务发现需求时
- Elasticsearch: 当有全文搜索/日志分析需求时
- Jenkins: 当有 CI/CD 自动化需求时

## Transferable Rule

**用最小实验验证理解。** 每学完一个技术，用最小可运行的项目验证"我能解释它解决什么问题"。不堆积名词，验证一个再学下一个。暂缓学习的技术不是不重要，而是等触发条件出现时再学更高效。
