## 1. 评分体系概述

### 1.1 设计理念

本评分体系的核心改进是**用视觉锚点代替数值测量**：

| 原有问题 | 解决方案 |
|---------|---------|
| "肘距躯干 > 20cm" | → "大臂与身体明显分离，像投掷动作" |
| "肩膀转动 > 90度" | → "背部面向球网，可见肩胛骨" |
| "肘角 > 160度" | → "手臂接近伸直，像握手距离" |

**评分者只需回答**: "看起来像什么？" 而不是 "测量出多少？"

### 1.2 六维度概览

| # | 维度 | 英文 | 子项数 | 评估内容 |
|---|------|------|--------|----------|
| 1 | 准备姿势 | Ready Position (RP) | 3 | 分腿步、重心、握拍 |
| 2 | 转体引拍 | Unit Turn (UT) | 4 | 肩髋转动、引拍路径、非持拍手 |
| 3 | 击球点 | Contact Point (CP) | 3 | 前后/左右位置、时机 |
| 4 | 动力链传递 | Kinetic Chain (KC) | 3 | 蹬地、顺序发力、大臂驱动 |
| 5 | 随挥恢复 | Follow-through (FT) | 3 | 幅度、方向、恢复速度 |
| 6 | 整体稳定 | Overall Stability (OS) | 4 | 一致性、拍面、握拍、平衡 |

**总计: 20 个子项**

### 1.3 Band 1-9 总体描述

| Band | 等级 | 总体描述 | 典型表现 |
|------|------|----------|----------|
| **9** | 专家级 | 职业级水平，几乎完美 | 像费德勒/纳达尔的慢动作回放 |
| **8** | 高级 | 熟练流畅，极少失误 | 像高水平业余比赛选手 |
| **7** | 熟练 | 动作规范，偶有小问题 | 像经过系统训练的俱乐部选手 |
| **6** | 胜任 | 基本结构完整，有明显局限 | 像学过一段时间的业余爱好者 |
| **5** | 中等 | 有框架但不稳定 | 像刚完成基础训练的初学者 |
| **4** | 有限 | 多个维度有问题 | 像自学没有教练指导的人 |
| **3** | 非常有限 | 严重问题，仅少量正确 | 像第一次拿拍的新手 |
| **2** | 间歇 | 几乎无法识别正确动作 | 偶尔碰巧做对，但无法持续 |
| **1** | 无效 | 无有效挥拍 | 无法识别任何网球技术元素 |

### 1.4 评分公式

```
维度 Band = round(该维度所有子项平均分)
整体 Band = round(六个维度 Band 的平均值)

如果触发任一"致命问题"：整体 Band = min(计算值, 2)
```

### 1.5 Core Issue 定义

**Core Issue** = 某个维度内评分 ≤ 4 的子项

- 每个维度最多报告 1 个 Core Issue（取最低分的子项）
- 如果该维度所有子项 > 4，则无 Core Issue
- `primaryCoreIssues` = 所有 Core Issues，按严重程度排序

## 2. 六个评分维度详解

### 2.1 准备姿势 (Ready Position)

**定义**: 球来之前的身体准备状态，为击球创造最佳条件

#### 子项定义

| 子项 | 英文 | 评估内容 |
|------|------|----------|
| 分腿步时机 | splitStepTiming | 对手击球瞬间的起跳和落地 |
| 重心准备 | readyStance | 膝盖弯曲、重心降低的程度 |
| 握拍准备 | gripReadiness | 握拍方式和准备姿态 |

#### Band 描述 - 分腿步时机 (splitStepTiming)

| Band | 视觉锚点 |
|------|----------|
| **9** | 对手击球瞬间双脚离地，落地时已判断来球方向，像弹簧一样弹起 |
| **7** | 有明显分腿步，时机基本正确，像准备接发球的运动员 |
| **5** | 有分腿步但时机偏晚，像反应慢半拍 |
| **3** | 分腿步不明显或几乎没有，像站着等球 |
| **1** | 完全站立不动，或无法识别准备动作 |

#### Band 描述 - 重心准备 (readyStance)

| Band | 视觉锚点 |
|------|----------|
| **9** | 膝盖明显弯曲，重心低，像准备起跑的运动员 |
| **7** | 膝盖有弯曲，重心较低，身体有弹性 |
| **5** | 膝盖轻微弯曲，但身体偏直立 |
| **3** | 几乎直立站着，像普通走路姿势 |
| **1** | 完全直立僵硬，或无法识别 |

#### Band 描述 - 握拍准备 (gripReadiness)

| Band | 视觉锚点 |
|------|----------|
| **9** | 标准正手握拍（东方/半西方/西方），非持拍手托住拍颈，像职业选手准备姿势 |
| **7** | 握拍正确，非持拍手有辅助，准备到位 |
| **5** | 握拍基本正确，但准备姿态不够规范 |
| **3** | 握拍不够标准，像握锤子或拍苍蝇 |
| **1** | 握拍完全错误（如乒乓球握法），或无法识别 |

### 2.2 转体引拍 (Unit Turn)

**定义**: 从准备姿势到引拍完成的转体动作

#### 子项定义

| 子项 | 英文 | 评估内容 |
|------|------|----------|
| 肩膀转动 | shoulderTurn | 肩膀从正对球网到侧身的程度 |
| 髋部转动 | hipTurn | 髋部与肩膀同步转动的程度 |
| 引拍路径 | backswingPath | 球拍向后引的轨迹和幅度 |
| 非持拍手 | nonRacketArm | 非持拍手的位置和作用 |

#### Band 描述 - 肩膀转动 (shoulderTurn)

| Band | 视觉锚点 |
|------|----------|
| **9** | 背部几乎完全对向球网，对手可以看到你的背号，像职业选手慢动作 |
| **7** | 明显侧身，肩膀线与球网成明显角度，像标准教科书姿势 |
| **5** | 有一定转动，但不够充分，像"半侧身" |
| **3** | 几乎没有转动，基本正对球网，像面对面打招呼 |
| **1** | 完全正对球网，或无法识别转体动作 |

#### Band 描述 - 髋部转动 (hipTurn)

| Band | 视觉锚点 |
|------|----------|
| **9** | 髋部与肩膀同步转动，形成完整的"上弦"蓄力，像拧紧的弹簧 |
| **7** | 髋部有明显转动，与肩膀基本同步 |
| **5** | 髋部有轻微转动，但明显少于肩膀，像只转上半身 |
| **3** | 髋部几乎不动，只有肩膀在转，像扭麻花 |
| **1** | 完全没有髋部转动，或无法识别 |

#### Band 描述 - 引拍路径 (backswingPath)

| Band | 视觉锚点 |
|------|----------|
| **9** | 流畅的环形引拍(C形或倒C形)，拍头先上后下形成loop，像画一个大弧线 |
| **7** | 有明显的引拍弧度，拍头有上升和下降 |
| **5** | 引拍较直线，缺少弧度，像直接往后拉 |
| **3** | 引拍幅度很小，球拍几乎没有向后移动 |
| **1** | 没有引拍，或无法识别引拍动作 |

#### Band 描述 - 非持拍手 (nonRacketArm)

| Band | 视觉锚点 |
|------|----------|
| **9** | 非持拍手向前指向来球方向，帮助瞄准和平衡，像指挥交通 |
| **7** | 非持拍手有向前伸展，起到一定平衡作用 |
| **5** | 非持拍手位置不固定，有时有用有时没用 |
| **3** | 非持拍手垂在身侧或乱摆，像不知道放哪里 |
| **1** | 非持拍手完全不参与，或姿势怪异 |

### 2.3 击球点 (Contact Point)

**定义**: 球拍与球接触时的位置、时机和身体状态

#### 子项定义

| 子项 | 英文 | 评估内容 |
|------|------|----------|
| 前后位置 | forwardPosition | 击球点相对于身体的前后位置 |
| 左右位置 | lateralPosition | 击球点与身体的横向距离 |
| 击球时机 | timing | 球被击中时的高度阶段 |

#### Band 描述 - 前后位置 (forwardPosition)

| Band | 视觉锚点 |
|------|----------|
| **9** | 击球点在身前舒适区，前脚的前方，像自然伸手就能够到 |
| **7** | 击球点在身前，基本在前脚位置附近 |
| **5** | 击球点有时偏后，接近身体正侧面 |
| **3** | 击球点经常在身侧或身后，像被球追着打 |
| **1** | 击球点混乱，无规律，或无法识别 |

#### Band 描述 - 左右位置 (lateralPosition)

| Band | 视觉锚点 |
|------|----------|
| **9** | 球距身体约一臂远，手臂自然伸展但不过度，像握手的舒适距离 |
| **7** | 距离基本合适，手臂有适当伸展 |
| **5** | 有时太近(挤着打)或太远(够着打) |
| **3** | 经常太近或太远，动作很别扭 |
| **1** | 位置混乱，或无法识别 |

#### Band 描述 - 击球时机 (timing)

| Band | 视觉锚点 |
|------|----------|
| **9** | 在球的上升期或最高点击球，节奏从容，像等着球送上门 |
| **7** | 时机基本正确，在合理的击球窗口内 |
| **5** | 时机不稳定，有时早有时晚 |
| **3** | 经常打晚球(球已下落)或匆忙击球 |
| **1** | 时机完全混乱，或无法判断 |

### 2.4 动力链传递 (Kinetic Chain)

**定义**: 从下肢到上肢的力量传递和顺序发力

#### 子项定义

| 子项 | 英文 | 评估内容 |
|------|------|----------|
| 蹬地发力 | groundForce | 后脚蹬地启动发力的程度 |
| 能量传递顺序 | sequencing | 腿→髋→躯干→肩→臂的发力顺序 |
| 大臂驱动 | upperArmDrive | 大臂参与挥拍的程度 |

#### Band 描述 - 蹬地发力 (groundForce)

| Band | 视觉锚点 |
|------|----------|
| **9** | 后脚有明显蹬地动作，像短跑起跑，能看到脚跟离地 |
| **7** | 后脚有推蹬，重心明显向前转移 |
| **5** | 有轻微蹬地，但不够明显 |
| **3** | 几乎没有蹬地，脚像钉在地上 |
| **1** | 完全没有腿部发力参与，或无法识别 |

#### Band 描述 - 能量传递顺序 (sequencing)

| Band | 视觉锚点 |
|------|----------|
| **9** | 明显的"鞭打"效应：腿先动、髋跟上、肩膀接力、最后手臂甩出，像多米诺骨牌 |
| **7** | 能看到下肢先启动、上肢跟进的顺序 |
| **5** | 有一定顺序，但不够流畅，像断断续续 |
| **3** | 顺序混乱，手臂先动或各部位同时动 |
| **1** | 完全没有动力链，各部位脱节，或无法识别 |

#### Band 描述 - 大臂驱动 (upperArmDrive)

| Band | 视觉锚点 |
|------|----------|
| **9** | 大臂像投标枪一样甩出，与身体有明显分离，肘部领先拍头 |
| **7** | 大臂明显参与，像慢速投球动作 |
| **5** | 大臂有一定参与，但主要靠小臂，像推门 |
| **3** | 大臂几乎不动，只有小臂在挥，像用手腕拨东西 |
| **1** | 大臂完全僵硬贴身，仅手腕动，或无法识别 |

### 2.5 随挥恢复 (Follow-through)

**定义**: 击球后的收拍动作和恢复准备姿势

#### 子项定义

| 子项 | 英文 | 评估内容 |
|------|------|----------|
| 随挥幅度 | followThroughRange | 击球后球拍继续挥动的距离 |
| 随挥方向 | followThroughPath | 收拍的方向和终点位置 |
| 恢复速度 | recoverySpeed | 回到准备姿势的速度 |

#### Band 描述 - 随挥幅度 (followThroughRange)

| Band | 视觉锚点 |
|------|----------|
| **9** | 完整的随挥，球拍一直挥到对侧肩膀上方或后方，像要把拍子甩出去 |
| **7** | 随挥充分，球拍越过身体中线，到达对侧 |
| **5** | 随挥不完整，击球后较快停止，像打了一半刹车 |
| **3** | 几乎没有随挥，击球后立即停止，像撞墙 |
| **1** | 完全没有随挥，或无法识别 |

#### Band 描述 - 随挥方向 (followThroughPath)

| Band | 视觉锚点 |
|------|----------|
| **9** | 球拍沿目标方向延伸后自然绕到对侧肩上，形成流畅弧线 |
| **7** | 收拍方向基本正确，到达对侧肩膀附近 |
| **5** | 收拍方向有偏差，有时太高或太低 |
| **3** | 收拍方向混乱，像随便甩一下 |
| **1** | 没有明确收拍方向，或无法识别 |

#### Band 描述 - 恢复速度 (recoverySpeed)

| Band | 视觉锚点 |
|------|----------|
| **9** | 随挥后立即恢复准备姿势，像职业选手连续击球 |
| **7** | 能较快恢复，有明显的回位意识 |
| **5** | 恢复较慢，有时站着看球飞走 |
| **3** | 恢复很慢，像完成一个动作后休息 |
| **1** | 没有恢复动作，或无法识别 |

### 2.6 整体稳定 (Overall Stability)

**定义**: 动作的一致性、控制能力和身体平衡

#### 子项定义

| 子项 | 英文 | 评估内容 |
|------|------|----------|
| 动作一致性 | consistency | 多次挥拍动作轨迹的相似程度 |
| 拍面控制 | racketFaceControl | 击球时拍面角度的稳定性 |
| 握拍稳定 | gripStability | 击球前后握拍的一致性 |
| 身体平衡 | bodyBalance | 整个动作过程中身体的稳定程度 |

#### Band 描述 - 动作一致性 (consistency)

| Band | 视觉锚点 |
|------|----------|
| **9** | 每次挥拍像复印机复制，动作轨迹几乎完全相同 |
| **7** | 动作基本一致，有轻微变化但不影响质量 |
| **5** | 一致性有限，能看出不同挥拍间的差异 |
| **3** | 每次挥拍都不一样，像在尝试不同动作 |
| **1** | 完全没有一致性可言，或无法识别 |

#### Band 描述 - 拍面控制 (racketFaceControl)

| Band | 视觉锚点 |
|------|----------|
| **9** | 击球瞬间拍面稳定，角度精确，像用尺子量过 |
| **7** | 拍面基本稳定，偶有轻微晃动 |
| **5** | 拍面有时翻转或不稳定 |
| **3** | 拍面经常翻转，角度不一致，像翻煎饼 |
| **1** | 拍面完全失控，或无法识别 |

#### Band 描述 - 握拍稳定 (gripStability)

| Band | 视觉锚点 |
|------|----------|
| **9** | 全程握拍稳定，击球瞬间无松动 |
| **7** | 握拍基本稳定，偶有轻微调整 |
| **5** | 挥拍过程中有明显调整或变化 |
| **3** | 握拍不稳定，经常松动或变换位置 |
| **1** | 握拍完全不稳定或错误，或无法识别 |

#### Band 描述 - 身体平衡 (bodyBalance)

| Band | 视觉锚点 |
|------|----------|
| **9** | 全程稳定如钟，击球后能立即做下一个动作 |
| **7** | 基本平衡，有良好的重心控制 |
| **5** | 有时失去平衡，需要额外步伐调整 |
| **3** | 经常失去平衡，像要摔倒 |
| **1** | 完全没有平衡可言，或无法识别 |

## 3. 致命问题检测

### 3.1 红线条件

**重要**: 以下任一条件为"是"，整体 Band 上限为 2，无论其他维度得分多高。

| # | 致命问题 | 视觉判断标准 |
|---|----------|-------------|
| 1 | **握拍完全错误** | 像乒乓球握法、握在拍喉、位置完全不对 |
| 2 | **全程仅小臂挥动** | 大臂完全贴着身体不动，只有小臂和手腕在动 |
| 3 | **全程僵硬** | 从头到尾像木头人，没有任何流畅性 |
| 4 | **动力链完全断裂** | 手臂与身体完全脱节，各自独立运动 |

### 3.2 触发后处理

```
如果 致命问题 >= 1 个为"是":
    整体 Band = min(原计算值, 2)
    标记: hasFatalIssue = true
    说明: "存在致命技术问题，需首先纠正基础"
```

### 3.3 致命问题检查表

评分时请逐项检查：

- [ ] 握拍方式是否正确？（不是乒乓球握法/羽毛球握法）
- [ ] 大臂是否参与挥拍？（不是只有小臂在动）
- [ ] 动作是否有流畅性？（不是全程僵硬）
- [ ] 身体是否协调？（手臂不是和身体完全脱节）

## 4. 评分流程

### 4.1 标准评分步骤

```
步骤 1: 观看视频，初步判断
    → 确认视频中有可识别的网球正手挥拍动作
    → 如果完全无法识别 → 整体 Band = 1，跳过后续步骤

步骤 2: 致命问题检测（优先）
    → 逐项检查 4 个红线条件
    → 如果任一为"是" → 标记 hasFatalIssue = true

步骤 3: 逐维度评分
    → 对 6 个维度的 20 个子项逐一评分 (Band 1-9)
    → 使用视觉锚点对照，选择最接近的描述
    → 计算每个维度的 Band = round(子项平均分)
    → 识别每个维度的 Core Issue（score ≤ 4 的子项）

步骤 4: 计算整体 Band
    → 整体 Band = round(六个维度 Band 的平均值)
    → 如果 hasFatalIssue = true → 整体 Band = min(计算值, 2)

步骤 5: 汇总 Core Issues
    → 列出所有 score ≤ 4 的子项
    → 按 score 从低到高排序
    → 确定 improvementPriority（最需要改进的子项）
```

### 4.2 评分注意事项

1. **必须使用视觉锚点** — 不要靠直觉，要对照描述
2. **不需要精确测量** — "看起来像什么"比"测出多少"更重要
3. **先做致命问题检测** — 这决定了分数上限
4. **如果犹豫，选较低分** — 宁可严格，避免高估
5. **单次视频看多遍** — 不同角度关注不同维度

### 4.3 维度 Band 计算示例

```
例: 转体引拍维度
- shoulderTurn: 5
- hipTurn: 4
- backswingPath: 6
- nonRacketArm: 5

维度 Band = round((5+4+6+5)/4) = round(5.0) = 5
Core Issue = hipTurn (score=4, ≤4)
```
