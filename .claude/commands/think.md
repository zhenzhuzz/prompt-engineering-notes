深度思考讨论模式：

## 模式说明

这是一个**只讨论、不执行**的模式。用于：
- 分析问题、提供见解
- 讨论方案、权衡利弊
- 深度思考、探索可能性

## 规则

1. **不执行任何操作**
   - 不创建文件
   - 不修改代码
   - 不运行命令
   - 只输出分析和建议

2. **使用 Thinking 模式**
   - 深度分析问题
   - 多角度思考
   - 提供有深度的见解

3. **输出格式**
   - 用 ASCII diagram 展示结构/流程/对比
   - 分点列出核心洞见
   - 给出可操作的建议（但不执行）

## 使用方式

```
/think 为什么 Agent 现在没必要？
/think 这个架构设计有什么问题？
/think 帮我分析这两个方案的优劣
```

## 与普通对话的区别

| 普通对话 | /think 模式 |
|---------|------------|
| 可能直接执行 | 只讨论不执行 |
| 快速给出答案 | 深度分析问题 |
| 面向任务 | 面向理解 |
