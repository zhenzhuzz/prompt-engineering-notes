高管简报格式 (Executive Brief):

生成一份高信息密度的文档，目标读者是高管/老板，需要在最短时间内理解项目全貌。

---

## 核心原则

- **结论先行**: 一句话概括 + 核心价值表格放最前
- **信息密度优先**: 3屏内看完核心内容
- **混合呈现**: Bullet Point + 表格 + ASCII Diagram
- **禁止代码块**: JSON 用字段路径表格，CSS 用属性-值表格

---

## Bullet Point vs 表格 使用场景

| 内容类型 | 推荐格式 | 原因 |
|----------|----------|------|
| 关键发现/结论 | ✅ Bullet | emoji 前缀醒目 |
| 操作步骤 | → 箭头链 | 线性流程清晰 |
| 技术选型 | **粗体**: 值 | 紧凑一行 |
| 特性列表 | - Bullet | 快速扫描 |
| 协议/接口 | 表格 | 多字段对比 |
| FAQ | 表格 | 问答对照 |
| 时间预估 | 表格 | 数值对比 |

## Bullet Point 格式规范

### 关键发现 (emoji 前缀)
```
**核心发现**:
✅ 诊断响应 < 2 分钟
✅ 单管理员吞吐量 20+/小时
⚠️ localStorage 限制 5-10MB
❌ 暂不支持多管理员
```

### 操作步骤 (箭头链)
```
**操作**: 输入名字 → 选视频 → 拖动选5秒 → 下一步 → 等待页
```

### 技术选型 (粗体+冒号)
```
**技术栈**: Vite + React 18 + TypeScript + localStorage (零后端)
```

### 特性列表 (短横线)
```
**数据资产**:
- **Atoms**: 46 个可诊断错误点
- **Boosts**: 22 个针对性练习
- **参考视频**: 8 个正确/错误示范
```

### 混合示例
```
**核心发现**:
✅ 1-2 分钟完整闭环
✅ 零后端依赖
⚠️ 同源限制 (同浏览器)

| 消息类型 | 方向 | Payload |
|----------|------|---------|
| MCQ_REQUEST | U→A | video, duration |
```

### 关键数字 (业务指标)
```
**关键数字**:
- **5** 个维度 (准备姿势、转体引拍、击球点、动力链、随挥)
- **46** 个 Atoms (可诊断错误点)
- **1** 个 Boost 推荐 (单一主矛盾策略)
```

**注意**: 关键数字应为业务指标，非技术指标
- ✅ 正例: 5 个维度、46 个错误点、1 个推荐
- ❌ 反例: 延迟 < 100ms、存储 5-10MB

### Step 详情 Bullet
```
## Step X: 标题

**触发**: 事件/消息名
**收到通知**: 🔔 音效 + 文案
**发送消息**: 消息名 (payload 简述)
**状态变化**: 状态A → 状态B
**音效**: 🔊 音效名 (触发时机)
**策略**: 💡 策略说明 (可选)

[ASCII 界面图]

**操作**: 步骤1 → 步骤2 → 步骤3
```

**字段说明**:
| 字段 | 用途 | 示例 |
|------|------|------|
| 触发 | 进入此步骤的条件 | 收到 MCQ_REQUEST |
| 收到通知 | 用户感知的提示 | 🔔 叮咚 + 语音 |
| 发送消息 | 此步骤产生的消息 | MCQ_SELECTED (5题) |
| 状态变化 | UI 状态转换 | 🔵 → ✅ 绿边框 |
| 音效 | 反馈音效 | 🔊 Complete |
| 策略 | 业务规则说明 | 💡 单一主矛盾 |
| 操作 | 用户操作箭头链 | 看视频 → 选题 → 发送 |

## 文档结构模板

1. **标题行**: `# 项目名 - 文档类型`
2. **元信息**: `**版本**: vX.X | **日期**: YYYY-MM-DD | **一句话**: 描述`
3. **核心价值表格**: 痛点 | 解决方案 | 价值 (3-5 行)
4. **关键发现**: ✅/⚠️/❌ Bullet Point (3-5 条)
5. **关键数字**: 业务指标 Bullet (3-5 条)
6. **业务流程图**: Phase 摘要 Bullet + ASCII 双列对照
7. **Step-by-Step**: Step 详情 Bullet + ASCII 界面图 + 箭头链操作
8. **技术附录**: 系统架构图、协议表格、数据资产 (放最后)
9. **FAQ 表格**: 问题 | 解答
10. **时间表格**: 阶段 | 时长 | 累计

## ASCII Diagram 规范

### 类型 A: 系统架构 (嵌套层级)
```
┌─────────────────────────────┐
│  顶层容器                    │
│  ┌─────────┐  ┌─────────┐   │
│  │ 组件 A  │──│ 组件 B  │   │
│  └────┬────┘  └────┬────┘   │
│       └─────┬──────┘        │
│             ▼               │
│       ┌─────────┐           │
│       │ 共享层  │           │
│       └─────────┘           │
└─────────────────────────────┘
```

### 类型 B: 业务流程 (双列对照)
```
┌──────────────────────────────────────────┐
│  Phase 1: 阶段名                          │
│  User 端              Admin 端            │
│  ┌────────┐           ┌────────┐         │
│  │ Step 1 │ ─MSG_A──► │ Step 2 │         │
│  │ Step 3 │ ◄─MSG_B── │ Step 4 │         │
│  └────────┘           └────────┘         │
└──────────────────────────────────────────┘
```

### 类型 C: 界面原型
```
┌─────────────────────────────────┐
│  Header                    [✕]  │
├─────────────────────────────────┤
│  ┌─────────────────────────┐    │
│  │   [组件区域]             │    │
│  └─────────────────────────┘    │
│  ┌────────┐  ┌────────┐        │
│  │ Btn A  │  │ Btn B  │   ← 🔊  │
│  └────────┘  └────────┘        │
└─────────────────────────────────┘
```

## 符号速查

| 类型 | 符号 |
|------|------|
| 结论 | ✅ ⚠️ ❌ 🚀 💡 |
| 方向 | → ← ↑ ↓ ► ◄ ▲ ▼ |
| 交互 | 🔔 🔊 👆 |
| 状态 | ✓ ✕ ☑ ☐ |
| 连接 | ── │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼ |

---

## 禁止元素

- ❌ 代码块 (仅 ASCII 图可用)
- ❌ 长段落 (超过 2 行)
- ❌ 过渡词 ("首先、其次、最后")
- ❌ 表格中的完整句子
- ❌ 冗余配置/启动说明
