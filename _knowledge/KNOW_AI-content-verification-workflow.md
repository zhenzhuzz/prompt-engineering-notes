# AI 内容校验工作流

> **核心问题**：AI 会"补充"它认为合理的内容，导致数据错误、经历杜撰。
> **解决方案**：显式约束 + 事后校验。

---

## 为什么 AI 会杜撰

```
┌─────────────────────────────────────────────────────────────────┐
│                 AI 的训练目标 vs 你的需求                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   AI 的训练目标：生成"连贯"的文本                                 │
│   你的需求：只说"确定"的事实                                      │
│                                                                 │
│   当你说"帮我写一篇文章"时：                                     │
│   AI 理解为：写一篇看起来合理的文章                              │
│   不是：只用你提供的事实写文章                                   │
│                                                                 │
│   结果：                                                         │
│   - 数字？编一个合理的                                           │
│   - 方法论？编一个常见的                                         │
│   - 经历？编一个典型的                                           │
│                                                                 │
│   AI 不会主动说"这里我不确定"，除非你要求它。                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**核心洞见**：AI 擅长"加工"，但不擅长"自我约束"。如果你想要准确性，必须**显式约束它**。

---

## 四种解决方案

### 方案 A：事实清单法（推荐长期使用）

```
步骤：
1. 让 AI 读完所有相关文档
2. 让 AI 输出"事实清单"（所有数字、时间线、方法）
3. 你确认/修正事实清单
4. AI 基于确认的事实创作
5. 任何不在清单里的内容，必须标注 [AI补充]

优点：你只需要检查 [AI补充] 部分
缺点：多一个步骤，但长期更高效
```

**Prompt 模板（事实提取）**：
```
请阅读我提供的文档，提取以下信息：

1. 所有具体数字（时间、数量、比例等）
2. 所有时间节点和时间线
3. 所有方法论和步骤
4. 所有观点和结论

输出格式：
- 每条信息标注来源文档和位置
- 如果有矛盾的信息，列出所有版本
```

---

### 方案 B：标注模式

**Prompt 模板（创作时标注）**：
```
创作内容时，使用以下标注：
- [文档] = 来自我提供的文档
- [推断] = 基于文档逻辑推断
- [补充] = 我没有提供，AI 补充的

每个具体数字、经历、方法论都必须标注来源。
```

---

### 方案 C：反向验证（推荐事后校验）

**Prompt 模板（创作后校验）**：
```
请逐段检查这篇文章，对每个具体信息进行验证：

检查项：
1. 每个具体数字（时间、数量、比例）
2. 每个具体经历或案例
3. 每个方法论或步骤
4. 每个观点或结论

输出格式：
| 内容 | 来源 | 状态 |
|------|------|------|
| [具体内容] | [文档名/位置] 或 [无来源] | [已验证/需确认/AI补充] |

特别标注：
- 找不到来源的内容，标注为 [无来源]
- 数字与文档不一致的，列出差异
```

---

### 方案 D：最小化创作（推荐快速发布）

```
不让 AI 创作"新内容"，只让它：
1. 从你的文档中提取要点
2. 重新排列结构
3. 优化表达方式

所有"核心内容"都由你提供。

AI 做：优化表达、调整结构、润色文案
AI 不做：补充数字、编造经历、创作新观点
```

---

## 完整 Prompt 模板（长期使用）

```markdown
# 写作约束

## 基本规则
1. 你只能使用我提供的文档中的信息
2. 不能添加任何我没有提供的数据、经历或方法论
3. 如果某个地方需要补充，必须用 [需要确认] 标注
4. 每个具体数字都要标注来源文档

## 标注规范
- [文档] = 来自我提供的文档
- [推断] = 基于文档逻辑推断
- [需要确认] = 我没有提供，需要你确认

## 输出要求
在文章末尾，列出：
1. 使用了哪些文档的哪些信息
2. 哪些内容是 [需要确认] 的
3. 有任何不确定的地方，主动提出
```

## 使用场景对照

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| 快速发布 | 方案 D | 你写核心，AI 只润色 |
| 重要内容 | 方案 A + C | 事实清单 + 事后校验 |
| 日常创作 | 方案 B | 边写边标注，效率适中 |
| 事后补救 | 方案 C | 已有内容，需要校验 |

---

## Transferable Rules

1. **AI 默认会"补充"**：不要假设 AI 只用你的信息
2. **显式约束比隐式期望有效**：在 prompt 里明确写出约束
3. **事后校验是必要的**：即使有约束，也要检查
4. **核心内容自己写**：AI 只负责优化表达，不负责创造事实
