## 1. 背景与目标

### 1.1 业务背景

用户在 Today 页面完成推荐训练后，或者想要自主选择练习内容时，需要一个集中的训练内容仓库。练习库是「Coach Yourself」理念的核心体现——用户是自己的教练，有完全的自主权决定练什么。

**场景痛点**:
- 用户想自己挑选练习但找不到入口
- 练习内容缺乏分类，难以快速定位
- 不知道练习的时长和目标

### 1.2 产品目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|--------|
| 快速找到目标练习 | 用户找到目标分类的时间 | < 10 秒 |
| 提高练习选择率 | 用户在 Drills 页面添加练习的比例 | >= 30% |
| 展示内容价值 | 用户浏览练习数量 | >= 5 个/次访问 |

### 1.3 成功指标

- **北极星指标**: 练习添加率（添加到今日训练的比例）
- **次级指标**:
  - 分类切换次数
  - 练习详情页访问率
  - 页面停留时长

## 3. 用户画像与旅程

### 3.1 目标用户

| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| 自主训练者 | 喜欢自己规划训练内容 | 快速浏览、自主选择 |
| 补充训练者 | 已有推荐但想加练 | 找到特定技术的练习 |
| 探索用户 | 想了解 Yoach 提供什么 | 浏览全部练习内容 |

### 3.2 核心用户旅程

```
用户进入 Drills Tab
    ↓
浏览分类标签栏
    ↓
选择目标分类（如「底线正手」）
    ↓
浏览该分类的练习列表
    ↓
查看练习详情 / 直接加入今日训练
    ↓
返回 Today 开始训练
```

### 3.3 边缘场景

| 场景 | 处理方式 |
|------|----------|
| 某分类无练习 | 显示空状态，引导查看全部 |
| 练习已加入今日 | 按钮显示「已加入」禁用状态 |
| 网络离线 | 显示本地缓存的练习 |

## 4. 功能需求

### 4.1 用户故事

| 编号 | 用户故事 | 验收标准 | 优先级 | 状态 |
|------|----------|----------|--------|------|
| US-DR01-01 | 作为用户，我想按技术类型浏览练习 | 1. 显示分类标签栏 2. 切换分类筛选列表 | P0 | 📋 |
| US-DR01-02 | 作为用户，我想了解每个练习的时长和目标 | 1. 卡片显示时长 2. 卡片显示目标描述 | P0 | 📋 |
| US-DR01-03 | 作为用户，我想一键将练习加入今日训练 | 1. 显示加入按钮 2. 点击后状态变更 | P0 | 📋 |
| US-DR01-04 | 作为用户，我想知道哪些练习已加入今日训练 | 1. 已加入显示禁用状态 2. 显示勾选图标 | P0 | 📋 |
| US-DR01-05 | 作为用户，我想探索所有可用练习 | 1. 默认显示全部 2. 可滚动浏览 | P0 | 📋 |

### 4.2 功能清单

#### F-DR01-01: 分类标签栏

| 分类 | 英文标识 | 说明 |
|------|----------|------|
| 全部 | all | 显示所有练习 |
| 底线正手 | forehand | 正手相关练习 |
| 底线反手 | backhand | 反手相关练习 |
| 发球 | serve | 发球相关练习 |
| 步伐 | footwork | 步伐移动练习 |
| 体能 | fitness | 体能训练练习 |

**显示规则**:
- 默认选中「全部」
- 选中状态：Teal 背景，白色文字
- 标签可水平滚动

#### F-DR01-02: 练习卡片

| 元素 | 说明 |
|------|------|
| 练习名称 | 主标题，17pt Semibold |
| 时长 | 右对齐，如「10 分钟」 |
| 目标描述 | 1-2 行描述 |
| 加入按钮 | 主按钮样式 |

#### F-DR01-03: 加入按钮状态

| 状态 | 按钮文字 | 样式 |
|------|----------|------|
| 未加入 | 加入今天训练 | 主按钮样式 |
| 已加入 | 已加入 | 禁用样式 + 勾选图标 |

#### F-DR01-04: 空状态

| 元素 | 内容 |
|------|------|
| 插图 | 空状态插图 |
| 标题 | 没有找到练习 |
| 副标题 | 试试其他分类 |
| 按钮 | 「查看全部练习」→ 切换到全部分类 |
