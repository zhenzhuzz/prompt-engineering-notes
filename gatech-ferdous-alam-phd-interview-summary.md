### 项目 2: 基于 CNN 的加工表面重建（学术项目）

**核心问题**: 如何从 1D 加速度信号重建 2D 表面形貌？

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    传统方法 vs AI 方法                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   传统方法:                                                                 │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   结构光/超声/红外相机 → 直接测量表面                                │   │
│   │   ❌ 问题: 制造环境恶劣，光学系统不可靠                              │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   AI 方法 (本项目):                                                         │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │   加速度信号 ──▶ CNN ──▶ 参数 (A, ω, φ) ──▶ 表面重建               │   │
│   │   ✅ 优势: 实时采集、环境鲁棒、高频信息丰富                          │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   为什么选择 CNN 而非 LSTM?                                                 │
│   ─────────────────────────────────────────────────────────────────────     │
│   • LSTM: 擅长序列预测（下一个词），不擅长特征提取                         │
│   • CNN: 自动特征提取能力强，适合高维映射                                  │
│   • 实验结果: CNN 效果最好                                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**可解释性设计**:

```
不是直接: 加速度信号 → 表面 (黑箱)

而是:     加速度信号 → CNN → 参数 (A, ω, φ) → 正弦波叠加 → 表面
                              ↑
                        可解释的物理参数
```

**振动能量量化**:
- 强迫振动 (Forced Vibration) - 蓝色
- 颤振 (Chatter Vibration) - 橙色
- 通过量化可识别主要振动类型，制定针对性抑制策略

**论文状态**: Published @ *Journal of Manufacturing Processes* (第一作者)

## Key Takeaways

### For PhD 申请者

1. **展示项目时区分工程 vs 学术**: 工程项目快速过，学术项目详细讲
2. **技术选择要能解释**: 如 "为什么选 CNN 而非 LSTM"
3. **承认不足但展示学习意愿**: "I'm more of an AI application user, but I'm willing to take fundamental AI courses"
4. **展示对研究方向的理解**: 提前了解教授研究，说明为什么 match

### For 面试准备

1. **准备 PPT/演示**: 申请人准备了 slides + 视频 demo
2. **预期技术追问**: 教授会问为什么选择某种技术方案
3. **准备反问问题**: 关于实验室方向、招生流程、学术 vs 创业定位

### For 跟进

1. **保持联系**: 任何进展都要通知教授
2. **关注时间节点**: 1 月是关键决策期
3. **表达持续兴趣**: 面试后适时 follow up
